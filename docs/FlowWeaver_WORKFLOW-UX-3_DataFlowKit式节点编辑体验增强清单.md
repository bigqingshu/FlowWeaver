# FlowWeaver WORKFLOW-UX-3：DataFlowKit 式节点编辑体验增强清单

## 目标

在不扩大后端协议、不改变工作流定义保存链路的前提下，将工作流编辑体验继续靠近 DataFlowKit 的高频操作：

- 复制节点。
- 批量删除节点。
- 上移/下移节点。
- 添加节点弹窗化。

本阶段只处理 Avalonia 桌面端的 workflow draft 编辑体验。最终保存仍通过现有 Workflow Definition revision 更新接口完成，后端继续作为最终校验边界。

## 当前基础

当前代码已经具备以下基础：

- `WorkflowDefinitionDraftNodePatcher.AddNode(...)` 支持新增节点，并支持插入到选中节点后。
- `WorkflowDefinitionDraftNodePatcher.DeleteNode(...)` 删除节点时会同步删除相关 `connections`。
- `WorkflowDefinitionDraftNodePatcher.MoveNode(...)` 支持调整 `nodes` 数组顺序。
- `WorkflowNodeListView` 已有节点列表、上移、下移、删除和高级 JSON 入口。
- `WorkflowAddNodeView` 当前是常驻表单，还不是弹窗。
- `connections` 已默认折叠为高级信息，不作为普通用户的主操作入口。

## 语义决策

### 复制节点

第一版复制节点只复制节点本身，不主动复制复杂连接。

复制内容：

- `node_type`
- `node_version`
- `display_name`
- `config`
- 可编辑输入字段对应的 draft JSON 结果

生成规则：

- 新节点必须生成唯一 `node_instance_id`。
- 建议命名为 `{原节点ID}_copy`，冲突时递增为 `{原节点ID}_copy_2`、`{原节点ID}_copy_3`。
- 默认插入到当前选中节点后。
- 复制后选中新复制节点。

连接规则：

- 第一版不复制原节点的上下游连接。
- 可复用现有新增节点的单下游自动接线能力，但只允许在线性、单下游、端口可推断的场景中发生。
- 多入、多出、复杂 DAG 场景不做自动连接推断。

失败边界：

- 未选择节点时不允许复制。
- draft JSON 非法时拒绝复制。
- 找不到目标节点时拒绝复制。
- 生成的新 `node_instance_id` 仍冲突时拒绝复制。

### 批量删除节点

第一版批量删除以“复选框选中集合”为基础，不依赖 ListBox 多选绑定。

删除规则：

- 删除所有已勾选节点。
- 同步删除所有 source 或 target 命中被删节点的 `connections`。
- 删除后清空批量选择集合。
- 如果当前选中节点被删除，清空当前节点选择；否则保留当前选中节点。

提示规则：

- 删除前第一版可以先不做阻断式二次确认。
- 删除完成后显示摘要：已删除节点数、同步移除连接数。
- 不要求用户逐条手动删除连接。

失败边界：

- 没有勾选节点时不允许批量删除。
- draft JSON 非法时拒绝删除并保留现有 draft 文本。
- 任一选中节点不存在时拒绝本次批量删除，避免部分成功造成认知混乱。

### 上移/下移节点

第一版上移/下移只调整 `nodes` 数组顺序，不改变 `connections`。

这意味着：

- 上移/下移影响节点列表显示顺序。
- 上移/下移影响高级 JSON 中 `nodes` 的排列。
- 上移/下移不改变运行依赖 DAG。
- 上移/下移不保证改变执行顺序。

后续如果要实现 DataFlowKit 式线性流程重排，应单独进入“线性模式自动重排连接”阶段，不混入 WORKFLOW-UX-3。

### 添加节点弹窗化

第一版将常驻新增节点表单改为弹窗或轻量浮层入口。

保留能力：

- 节点定义下拉选择。
- 手动编辑 `node_instance_id`。
- 手动编辑 `display_name`。
- 手动编辑 config JSON。
- 刷新节点定义入口。
- 原有 AddNode 命令、校验和错误展示逻辑。

入口规则：

- 节点列表操作区提供“新增节点”按钮。
- 弹窗打开后显示插入位置：
  - 有选中节点：插入到当前选中节点后。
  - 无选中节点：追加到末尾。
- 新增成功后关闭弹窗并选中新节点。
- 新增失败时保留弹窗和输入内容。

不做内容：

- 不引入自由画布。
- 不引入拖拽连线。
- 不恢复固定节点目录窗口。
- 不在弹窗内实现复杂 schema 表单编辑。

## 分阶段实施顺序

### WORKFLOW-UX-3.0：语义清单

产出本文档。

验收：

- 文档明确复制节点、批量删除、上移/下移、添加节点弹窗化边界。
- 明确哪些能力不在第一版实现。
- 不改代码。

### WORKFLOW-UX-3.1：复制节点 patcher

范围：

- 在 `WorkflowDefinitionDraftNodePatcher` 增加复制节点能力。
- 只实现 draft JSON 层，不接 UI。

验收：

- 复制节点生成唯一 ID。
- config 深拷贝。
- 新节点插入到原节点后。
- 原节点连接不被错误复制。
- 非法 JSON、缺失节点、重复 ID 等失败边界有测试。

### WORKFLOW-UX-3.2：复制节点 UI 接入

范围：

- `MainWindowViewModel` 增加复制节点命令。
- `WorkflowNodeListView` 节点操作区增加复制按钮。
- 补充中英文文案。

验收：

- 未选择节点时复制按钮不可用。
- 有选中节点且 draft 可编辑时复制按钮可用。
- 复制后选中新节点。
- 失败时保留 draft。

### WORKFLOW-UX-3.3：批量选择前置模型

范围：

- 为工作流节点列表增加批量选择状态。
- 先只显示勾选和选中计数，不执行删除。

验收：

- 勾选状态随节点列表刷新保持合理同步。
- 节点被删除或 draft 重新加载后，失效勾选项被清理。
- 结构测试覆盖批量选择入口。

### WORKFLOW-UX-3.4：批量删除

范围：

- 增加批量删除 patcher 或复用安全的批量删除逻辑。
- ViewModel 接入批量删除命令。

验收：

- 删除多个节点。
- 同步删除相关 connections。
- 删除后提示节点数和连接数。
- 没有勾选节点时按钮不可用。

### WORKFLOW-UX-3.5：上移/下移语义收口

范围：

- 明确 UI 文案和测试中的上移/下移语义。
- 不改变现有运行依赖。

验收：

- 文案避免暗示执行顺序已改变。
- 测试证明移动节点时 connections 保持不变。

### WORKFLOW-UX-3.6：添加节点弹窗化

范围：

- 将常驻新增节点表单改为弹窗/浮层入口。
- 尽量复用现有 `WorkflowAddNodeView` 内容和命令绑定。

验收：

- 打开弹窗后可新增节点。
- 新增成功关闭弹窗。
- 新增失败保留弹窗与输入。
- 刷新节点定义入口仍可用。

### WORKFLOW-UX-3.7：总体验收复核

范围：

- 汇总阶段完成情况。
- 运行 Avalonia 相关测试。
- 补充阶段记录。

验收：

- 复制、批量删除、移动、弹窗添加均有测试覆盖。
- 不影响预览选中节点、运行工作流、保存 revision 等既有链路。
- 未实现的线性自动重排连接作为后续阶段保留。

## 第一版明确不支持

- 自由画布式节点编排。
- 拖拽连线。
- 多入多出 DAG 自动重排。
- 上移/下移自动改写 `connections`。
- 复制复杂连接拓扑。
- 节点配置 schema 表单化编辑。
- 撤销/重做。

## 后续保留方向

如果第一版体验稳定，后续可以单独评估：

- 线性模式自动重排连接。
- 复制节点时复制简单线性连接。
- 批量操作撤销。
- 节点拖拽排序。
- 节点配置 schema 表单化。
- 工作流模板与片段复用。
