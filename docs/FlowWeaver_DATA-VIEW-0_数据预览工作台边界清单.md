# FlowWeaver DATA-VIEW-0 数据预览工作台边界清单

日期：2026-07-06

## 目标

新增独立的“数据预览”工作台，用于查看、搜索、复制、编辑和受控保存工作流产生的表数据，并为后续 SQL 表管控能力建立稳定边界。

本阶段只固化边界与执行顺序，不直接修改代码。

## 与现有“数据”页的关系

现有“数据”页保留，不删除。

两个页面职责区分如下：

| 页面 | 主要职责 | 第一版定位 |
| --- | --- | --- |
| 数据 | TableRef、SharedPublication、共享发布版本等数据资产摘要 | 工作流之间数据共享和运行产物追踪 |
| 数据预览 | 真实表格内容查看、搜索、复制、编辑和保存 | 面向用户的数据检查与整理工作台 |

因此“数据”页偏数据资产和共享关系，“数据预览”页偏表格操作。

## 当前已具备能力

后端已经具备只读数据基础：

- `GET /api/v1/data/{table_ref_id}`：读取 TableRef。
- `GET /api/v1/data/{table_ref_id}/schema`：读取字段结构。
- `GET /api/v1/data/{table_ref_id}/summary`：读取摘要和行数。
- `GET /api/v1/data/{table_ref_id}/rows`：分页读取真实行数据。

Avalonia API 客户端已经具备对应方法：

- `GetTableDataSchemaAsync`
- `GetTableDataSummaryAsync`
- `GetTableDataRowsAsync`

当前工作流页面已有快速数据预览能力，但它主要服务“选中节点后快速看运行结果”，不适合作为完整表格整理工作台。

## 用户目标

新增左侧导航按钮：

```text
工作流
数据预览
运行
数据
日志
设置
```

“数据预览”页面目标：

- 左侧列出可预览表来源，可参考现有“数据”页的表引用窗口。
- 选中表后，右侧显示真实表格。
- 支持下拉或列表切换同一来源下的多张表。
- 支持查看和修改单元格值。
- 支持保存，后续用于管控 SQL 表、运行表和内存表。
- 支持数据搜索。
- 支持类似 DataFlowKit 的 xlsx/剪贴板表格复制解析能力。
- 工作流页快速预览区域增加“显示详情”按钮，点击后跳转到“数据预览”，并定位对应节点输出表。

## 核心边界

### 1. 不替代工作流页快速预览

工作流页的数据预览继续保留，适合快速确认当前选中节点输出。

“显示详情”只负责跳转和定位，不把复杂编辑功能塞回工作流页。

### 2. 不直接编辑已发布运行结果

工作流运行产生的 `PUBLISHED` TableRef 应默认视为不可变运行结果。

第一版编辑建议在 UI 内形成本地草稿：

```text
TableRef rows
-> UI 本地编辑草稿
-> 保存为新表 / 新 TableRef / 明确 SQL 写回
```

不建议直接覆盖原 TableRef 对应的运行结果。

### 3. SQL 支持必须走后端受控边界

支持 SQL 是目标，但 UI 不应直接连接 SQLite 或其他数据库。

SQL 读取和保存应走后端 API，由后端根据 `TableRef.provider_id`、`storage_kind`、`capabilities` 和保存策略判断是否允许。

推荐能力声明：

| 能力 | 含义 |
| --- | --- |
| `READ` | 允许读取 rows/schema/summary |
| `APPEND` | 允许追加行 |
| `WRITE` | 允许受控写入 |
| `PATCH_CELL` | 允许单元格级更新 |
| `REPLACE` | 允许整表替换 |
| `SAVE_AS` | 允许另存为新表或新 TableRef |

当前已存在 `READ` 和部分运行表 `APPEND` 概念；其余能力应在后续阶段谨慎补齐。

### 4. 保存策略分层

保存不能只有一个“保存”按钮直接落库，需要区分目标：

| 保存方式 | 第一版建议 | 说明 |
| --- | --- | --- |
| 保存为新运行表 | 优先支持 | 不破坏原运行结果 |
| 保存为内存表 | 可支持 | 适合临时整理和后续节点使用 |
| 保存为 SQL 新表 | 支持目标 | 需要后端 SQL save API |
| 覆盖 SQL 原表 | 后置支持 | 必须有能力声明、确认、事务和备份策略 |
| 覆盖已发布 TableRef | 不建议 | 会破坏运行可追溯性 |

### 5. DataFlowKit 能力迁移边界

可以参考 DataFlowKit 的能力：

- 分页表格预览。
- 搜索和定位。
- 剪贴板表格解析。
- xlsx 表格复制解析。
- 单元格编辑。
- 保存到 SQLite 表。
- 导出或另存。

但 FlowWeaver 不应复制 DataFlowKit 的 Tk/UI 内部实现。应抽象成后端 API + Avalonia MVVM 状态：

```text
API/服务：负责读取、解析、保存和校验
ViewModel：负责状态、脏标记、分页、选择和错误提示
View：负责表格、工具栏和轻量交互
```

```text
┌────────────────────────────────────────────────────────────┐
│ 数据预览                                                    │
├───────────────────┬────────────────────────────────────────┤
│ 来源/表列表         │ 工具栏：刷新 搜索 复制 粘贴解析 保存      │
│                   ├────────────────────────────────────────┤
│ 工作流运行          │ 表格元信息：来源、TableRef、行数、状态     │
│ 节点输出            ├────────────────────────────────────────┤
│ TableRef           │ 表格内容                                │
│ SharedPublication  │                                        │
│ SQL 映射表          │                                        │
└───────────────────┴────────────────────────────────────────┘
```

## 工作流页跳转语义

工作流页快速预览区域新增“显示详情”：

```text
当前选中 workflow node
-> 当前或最近 run 中找到对应 NodeRun
-> 找到该 NodeRun 的可读 TableRef
-> 切换 Shell 到 数据预览
-> 自动选择该 TableRef 并加载 rows
```

如果没有可读表：

- 停留在工作流页或跳转后展示空状态均可。
- 提示“当前节点尚无可预览表”。
- 不自动运行工作流。

## DATA-VIEW 执行顺序

### DATA-VIEW-1：新增数据预览导航与空页面

- 新增 Shell 页 key 和 descriptor。
- 新增 `DataPreviewPage` 空页面。
- 新增本地化文本。
- 测试导航清单和页面 key。

### DATA-VIEW-2：TableRef/SQL 只读表格加载

- 复用或抽取 TableRef 列表模型。
- 选中 TableRef 后调用 rows/schema/summary。
- 显示表格列和分页 rows。
- 支持 `RUNTIME_SQL` 和 `EXTERNAL_SQL` 只读预览。
- 不保存、不编辑。

### DATA-VIEW-3：工作流页显示详情跳转

- 工作流快速预览区域增加“显示详情”。
- 将当前预览来源传递到数据预览页。
- 自动切换左侧导航并加载目标 TableRef。

### DATA-VIEW-4：搜索、分页和复制

- 增加搜索框。
- 第一版可先做当前页客户端搜索。
- 增加上一页、下一页、刷新。
- 增加复制选中单元格/行/表格为 TSV。
- 不改变后端。

### DATA-VIEW-5：剪贴板 / xlsx 表格解析导入

- 先支持剪贴板 TSV/CSV 风格解析。
- xlsx 复制内容通常会进入剪贴板 TSV，可先按 TSV 解析。
- 后续如需直接读 `.xlsx` 文件，再单独做文件导入。
- 解析结果进入本地临时草稿表，不立即保存。

### DATA-VIEW-6：编辑脏状态和另存边界

- 表格单元格可编辑。
- 编辑后显示 dirty 状态。
- 支持撤销到最后一次加载值。
- 保存按钮第一版只进入“另存为”策略，不覆盖原 TableRef。

### DATA-VIEW-7：SQL 受控保存策略

- 后端新增最小保存 API 前先确认能力声明。
- 保存到 SQL 新表优先于覆盖原表。
- 覆盖原 SQL 表必须有显式确认、事务、失败回滚和可选备份。
- UI 根据后端返回的 capabilities 控制按钮启用。

## 第一轮验收基线

DATA-VIEW-0 到 DATA-VIEW-7 完成后，应满足：

- 左侧有独立“数据预览”入口。
- 可从工作流页跳转到节点输出表详情。
- 可在数据预览页查看真实表格 rows。
- 可分页、搜索、复制表格内容。
- 可从剪贴板解析表格到临时草稿。
- 编辑后能明确看到未保存状态。
- 保存策略不会破坏原工作流运行结果。
- SQL 支持通过后端受控能力实现，而不是 UI 直接写数据库。

## 暂不做

- 不删除现有“数据”页。
- 不把 SharedPublication 管理入口迁入数据预览页。
- 不做拖拽画布。
- 不做完整 Excel 文件编辑器。
- 不在 UI 中直连数据库文件。
- 不绕过后端 provider/capabilities 直接写 SQL。
- 不默认覆盖已发布运行结果。
