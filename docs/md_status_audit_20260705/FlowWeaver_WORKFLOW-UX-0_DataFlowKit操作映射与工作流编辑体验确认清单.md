# FlowWeaver WORKFLOW-UX-0：DataFlowKit 操作映射与工作流编辑体验确认清单

> 未实现：完整 DataFlowKit 操作等价、真实业务节点迁移、复杂表格操作映射和一体化视觉编排未实现。

> 当前阶段：WORKFLOW-EDIT-3.0 后的工作流 UX 方向分析
> 适用范围：Avalonia 工作流页布局、节点列表式编辑、数据预览入口、后续预览/运行到节点能力

## 1. 背景

FlowWeaver 的底层架构已经具备工作流定义、revision、运行、节点执行、TableRef、SharedPublication、事件和审计等正式路径。

当前 Avalonia 工作流页仍偏工程调试形态：

```text
左侧：工作流列表
中间：工作流详情、节点、连接、结构化编辑表单
右侧：Draft JSON、节点目录
```

该布局有利于验证底层协议，但不利于快速完成 DataFlowKit 主程序中常用的工作流操作，例如复制节点、批量删除节点、预览到目标节点、运行到选中节点、上移下移等。

本阶段目标是先确认 UX 方向：是否将工作流页主体验调整为“节点顺序列表 + 快速操作 + 数据预览”，同时把连接和 JSON 降级为高级/调试能力。

## 2. 与当前计划的关系

该方向与当前计划不冲突，但需要作为新阶段处理。

原因：

```text
WORKFLOW-EDIT-2/3 已完成最小结构化编辑和布局收口
当前计划明确不直接扩 port/schema/画布
本方向先做列表式工作流编辑，不直接进入自由画布
connections 底层模型继续保留，只调整 UI 暴露方式
```

需要遵守的既有边界：

```text
保存仍走 WorkflowDefinitionDraftJson
保存前仍需要 Validate 通过
revision 冲突语义不改变
node_type / node_version / node_instance_id 仍是机器标识
connections 仍是执行 DAG 的事实来源
```

## 3. 建议的产品方向

建议将工作流页改成更接近 DataFlowKit 的“顺序式编辑体验”：

```text
工作流列表
→ 当前工作流节点顺序列表
→ 选中节点配置/操作区
→ 下方数据预览
```

其中：

```text
节点顺序列表是主界面
连接信息默认隐藏或折叠
Draft JSON 作为高级查看/高级编辑保留
节点目录不再固定占据右下角，改为“添加节点”弹窗/抽屉/搜索下拉
数据预览作为长条区域固定在工作流页下方或运行/数据页下方
```

## 4. 用户提出的改动逐项分析

| 改动 | 建议 | 是否冲突 | 说明 |
| --- | --- | --- | --- |
| 复制节点 | 支持 | 不冲突 | 作为 draft mutation，复制 config 并生成新 node_instance_id |
| 批量删除节点 | 支持 | 不冲突 | 需要删除关联 connections，并在执行前显示影响摘要 |
| 上移/下移 | 支持，但需限定语义 | 不冲突 | 线性模式下可自动重排连接；DAG 模式下只调整显示顺序 |
| 预览到目标节点 | 支持，但后端需新增能力 | 需要新阶段 | 本质是 preview_until_node，需要执行目标节点的上游闭包 |
| 运行预览到选中节点 | 支持，但后端需新增能力 | 需要新阶段 | 本质是 run_until_node，需要副作用边界 |
| 工作流窗口连接块去掉 | UI 可收起，模型不可删除 | 不冲突 | connections 仍保留为高级/调试视图 |
| 草稿 JSON 窗口改为节点顺序显示 | 建议 | 不冲突 | JSON 保留为高级入口，主界面显示节点顺序 |
| 添加节点操作按钮放到节点顺序区 | 建议 | 不冲突 | 节点目录改为弹窗/抽屉或搜索选择 |
| 节点目录固定窗口删除 | UI 可删除 | 不冲突 | 后端 node-definitions 和前端加载能力仍保留 |
| 工作流外其他窗口下方加数据预览 | 建议抽成统一组件 | 不冲突 | 可先做 TableRef 行数据只读预览 |

## 5. 关键语义：线性模式与 DAG 模式

DataFlowKit 主程序更接近顺序式工作流，因此“上移/下移”“运行到某个节点”比较自然。

FlowWeaver 当前底层是 DAG 模型，执行顺序由 connections 决定。因此需要明确两种 UX 模式：

### 5.1 线性编辑模式

适合第一版快速补齐 DataFlowKit 常用操作。

```text
节点列表顺序 = 用户主要编辑对象
系统自动维护相邻节点连接
添加节点默认插入到当前选中节点后
上移/下移会同步重排相邻连接
删除节点会自动衔接前后节点或提示断开
```

线性模式建议作为默认体验。

### 5.2 高级 DAG 模式

适合未来多输入、多输出、分支、合并、共享表、多源读取场景。

```text
connections 作为显式编辑对象
允许一个节点连接多个下游
允许多个上游输入同一个节点
上移/下移只影响显示顺序，不改变执行 DAG
```

高级 DAG 模式不建议在 WORKFLOW-UX-0 后立即实现。

## 6. 推荐布局草案

第一版建议：

```text
┌─────────────────────────────────────────────────────────────┐
│ 工作流列表                                                   │
├─────────────────────────────────────────────────────────────┤
│ 当前工作流：名称 / revision / 状态 / Validate / Save / Run   │
├─────────────────────────────────────────────────────────────┤
│ 节点顺序列表                                                 │
│ [添加] [复制] [删除] [批量删除] [上移] [下移] [预览到此]       │
│                                                             │
│ 1. 生成测试表       GenerateTestTableNode                    │
│ 2. 过滤行           FilterRowsNode                           │
│ 3. 发布共享表       PublishSharedTablesNode                  │
├─────────────────────────────────────────────────────────────┤
│ 选中节点配置 / 摘要 / JSON 兜底                              │
├─────────────────────────────────────────────────────────────┤
│ 数据预览                                                     │
└─────────────────────────────────────────────────────────────┘
```

说明：

```text
连接列表默认不出现
Draft JSON 不作为主编辑面板
节点目录通过“添加节点”入口调用
数据预览作为底部长条区域
```

## 7. 后端能力缺口

以下能力可以只在前端 draft 层完成：

```text
复制节点
批量删除节点
上移/下移的显示顺序
节点目录弹窗化
Draft JSON 高级折叠
连接块隐藏/折叠
```

以下能力需要后端或执行协议支持：

```text
TableRef 行数据预览 API
preview_until_node
run_until_node
副作用节点在预览模式下的处理策略
预览结果与正式运行结果的隔离
运行到节点时 RuntimeEvent / NodeRun 的表达
```

## 8. 数据预览建议

建议新增统一的数据预览区域，而不是每个页面各写一套。

候选组件：

```text
DataPreviewPane
```

第一版能力：

```text
选中 TableRef 后显示前 100 行
显示字段名、数据类型和行数摘要
只读
不导出
不编辑
不做复杂筛选
```

后续可扩展：

```text
搜索
导出 xlsx
按列过滤
SharedPublication member 跳转
NodeRun 输出表跳转
```

## 9. 建议执行顺序

### WORKFLOW-UX-0：方向确认

只写文档，确认：

```text
工作流页是否改为节点顺序列表优先
连接是否默认收起
Draft JSON 是否降级为高级入口
节点目录是否改为添加节点入口
数据预览是否作为底部长条区域
线性编辑模式是否作为第一版默认体验
```

### WORKFLOW-UX-1：布局原型

只改 Avalonia View 层为主：

```text
移除固定节点目录区域
Draft JSON 改为折叠/高级区域
节点列表区域增大
连接区域默认折叠
预留底部 DataPreviewPane 占位
```

### WORKFLOW-UX-2：节点操作命令

先做纯 draft mutation：

```text
复制节点
批量删除节点
上移/下移显示顺序
操作后校验失效
保存仍要求 Validate 成功
```

### WORKFLOW-UX-3：数据预览只读闭环

后端和前端都需要小步：

```text
GET /api/v1/table-refs/{table_ref_id}/rows
Avalonia TableRef rows DTO
DataPreviewPane 显示前 100 行
```

### WORKFLOW-UX-4：运行到节点 / 预览到节点边界分析

先分析不实现：

```text
副作用节点策略
预览 run 与正式 run 的隔离
NodeRun 状态表达
RuntimeEvent 表达
TableRef 生命周期
```

## 10. 明确不建议现在做

当前不建议直接进入：

```text
自由拖拽画布
拖拽连线
完整 JSON Schema 动态表单
复杂字段选择器
插件 UI 模块系统
运行中热加载 UI 模块
多分支 DAG 可视化编辑器
```

原因：

```text
这些能力会扩大范围
会绕开当前最急需的 DataFlowKit 常用操作迁移
会增加与后端 schema、port、validation、执行副作用边界的耦合
```
