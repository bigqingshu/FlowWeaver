# 1. 第一阶段技术栈

## 1.1 运行环境

```text
主要操作系统：Windows 11
兼容目标：Windows 10
Python：3.12.x
字符编码：UTF-8
进程启动方式：spawn
CPU架构：x86-64
```

第一阶段不要求：

```text
Linux服务化部署
macOS支持
Docker部署
多机运行
```

## 1.2 依赖管理

```text
依赖管理：uv
项目配置：pyproject.toml
虚拟环境：.venv
锁定文件：uv.lock
```

禁止把虚拟环境、运行数据库、日志和临时文件提交到Git。

## 1.3 核心后端

```text
语言：Python 3.12
API框架：FastAPI
ASGI服务器：Uvicorn
外部控制接口：HTTP REST + WebSocket
数据模型：Pydantic v2
内部元数据库：SQLite
ORM / SQL构建：SQLAlchemy 2.x
数据库迁移：Alembic
```

说明：

- EngineHost只监听本机地址；
- 第一阶段默认绑定`127.0.0.1`；
- 桌面UI也通过HTTP和WebSocket连接EngineHost；
- 后续增加Web管理端时复用同一套公共API；
- 不允许桌面UI直接调用EngineHost内部Python对象。

## 1.4 内部进程通信

```text
进程管理：multiprocessing
启动上下文：multiprocessing.get_context("spawn")
传输通道：multiprocessing.connection.Connection
消息编码：MessagePack
消息发送：send_bytes / recv_bytes
```

内部IPC禁止直接发送任意Python对象或依赖pickle序列化。

所有IPC消息必须：

```text
由Pydantic模型定义
转换为纯字典
使用MessagePack编码
带protocol_version
带message_id
带timestamp
```

## 1.5 桌面UI

```text
UI框架：PySide6
HTTP客户端：httpx
WebSocket客户端：websockets
网络线程：独立QThread
```

第一阶段UI只实现最小管理界面，不实现复杂节点画布。

## 1.6 日志与审计

```text
普通日志：Python logging
跨进程日志：QueueHandler + QueueListener
日志文件：JSON Lines
审计存储：SQLite audit_events表
日志轮转：RotatingFileHandler
```

普通日志与审计分开：

```text
普通日志：
用于调试、异常、运行状态

审计事件：
用于记录工作流、节点、数据和权限操作
```

## 1.7 测试

```text
测试框架：pytest
异步测试：pytest-asyncio
UI测试：pytest-qt
超时测试：pytest-timeout
覆盖率：pytest-cov
```

第一阶段最低覆盖率：

```text
核心协议、状态机、Registry、Supervisor：80%
整体项目：70%
```

# 3. 全局工程规则

## 3.1 ID规则

全部ID使用字符串UUIDv7；若库不可用，第一阶段允许UUID4。

命名示例：

```text
workflow_id
workflow_run_id
node_instance_id
node_run_id
executor_id
table_ref_id
publication_id
input_snapshot_id
lease_id
message_id
event_id
```

禁止使用自增整数作为跨模块公共ID。

## 3.2 时间规则

```text
内部时间：UTC
传输格式：ISO 8601
UI显示：转换为本地时区
```

## 3.3 枚举规则

所有状态和模式使用字符串枚举，不使用裸整数。

示例：

```python
class NodeRunStatus(str, Enum):
    PENDING = "PENDING"
    READY = "READY"
    RUNNING = "RUNNING"
    SUCCEEDED = "SUCCEEDED"
    FAILED = "FAILED"
```

## 3.4 错误规则

所有跨进程、跨API错误统一为：

```text
error_code
message
details
retryable
origin
trace_id
```

禁止只传异常字符串。

# 4. 进程模型

```text
EngineHost
├─ FastAPI/Uvicorn
├─ RuntimeStore
├─ RuntimeDataRegistry
├─ PermissionManager
├─ EventRouter
├─ Supervisor
└─ WorkflowRunProcess N
   ├─ DAGScheduler
   ├─ NodeTaskManager
   ├─ NodeExecutorProcess 1
   ├─ NodeExecutorProcess 2
   └─ DedicatedWorker（预留）
```

第一阶段规则：

```text
EngineHost不得执行普通节点业务逻辑
WorkflowRunProcess不得长时间执行普通业务节点
普通节点默认进入NodeExecutorProcess
每个WorkflowRunProcess默认启动2个NodeExecutorProcess
执行器数量可配置，但第一阶段不动态伸缩
```

第一阶段允许当前工作流对其`MANAGED_RUNTIME`运行数据库拥有默认完整内部权限，但仍必须产生权限句柄和审计事件。

```python
class Supervisor:
    def start_workflow_process(
        self,

    ) -> str:
        ...

    def stop_workflow_process(
        self,
        workflow_run_id: str,

        ...

    def start_executor(
        self,
        workflow_run_id: str,
    ) -> str:
        ...

        self,
        executor_id: str,
        reason: str,
    ) -> None:
        ...

    def restart_executor(
        self,
        workflow_run_id: str,
        old_executor_id: str,
    ) -> str:
        ...
```

默认时间：

```text
执行器心跳间隔：2秒
执行器心跳失联阈值：8秒

执行器强制终止后重建：立即
WorkflowRunProcess心跳间隔：2秒
WorkflowRunProcess失联阈值：10秒
```

节点超时流程：

```text
TIMED_OUT
→ 发送取消请求
→ 等待5秒

→ 清理STAGING
→ 重建执行器
→ 根据失败策略处理下游
```
