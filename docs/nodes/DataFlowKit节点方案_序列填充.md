# 节点方案：FillSequenceNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.sequence_fill` / 序列填充

FlowWeaver 暂定类型名：`FillSequenceNode`

节点方向：数据处理 / 序列生成

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

为指定字段按范围生成递增、递减或格式化序列值。

用户看到的能力：配置起始值、步长、范围和格式，运行后得到带序列的新表。

不解决的内容：不生成全局唯一 ID，不跨运行保证连续编号。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出字段和行数通常不变，只改变目标字段或目标范围。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `target_field` | 目标字段 |
| `start_row` | 起始行 |
| `direction` | 填充方向 |
| `start_value` | 序列起始值 |
| `step` | 步长 |
| `count_source_mode` | 使用结束条件或固定数量 |
| `end_mode` / `count` / `end_row` / `reference_field` | 范围控制 |
| `overwrite_rule` | 覆盖规则 |
| `zero_pad` | 补零位数 |
| `prefix` / `suffix` | 前缀和后缀 |

## 数据契约

节点在目标范围内生成序列文本或数字。

第一版不承诺跨 NodeRun 的连续性，序列只与本次输入和配置有关。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要。

## 主程序交互边界

节点不向主程序申请全局序列号。

若未来需要全局唯一 ID，应另起节点规划。

## 运行记录

结果摘要建议包含目标字段、起始值、步长、写入数量、输出格式。

失败提示应指出数字格式非法、范围越界或覆盖规则冲突。

## 验收方式

固定数量和按参考字段边界两种范围均可生成序列。

补零、前缀、后缀格式正确。

只填空单元格策略正确。

## 实现前置依赖

需要通用表格节点执行入口。

需要范围解析 helper。

节点名称：序列填充。

节点定位：数据处理 / 序列生成。

优先级：P1。

当前状态：规划中，代码未实现。

要解决的问题：见上文“要解决的问题”章节。

用户看到的能力：见上文“用户看到的能力”描述。

第一版不解决的内容：见上文“不解决的内容”描述。

注册参数：
- node_type：FillSequenceNode
- node_version：1.0
- plugin_id：core
- provider_type：builtin
- category：数据处理
- ui_visibility：visible
- enabled：规划期为 false；实现和验收后再按节点成熟度设为 true。
- display_name：序列填充
- config_schema：沿用本文“配置项草案”，后续落到统一 config_schema。
- input_ports：一个标准 TableRef 输入。
- output_ports：一个新的标准 TableRef 输出。
- implementation_ref：builtin.FillSequenceNode（暂定内部执行入口，后续实现时绑定真实实现；不对普通 UI 暴露）。

输入说明：见上文“输入输出”章节；第一版按 input_ports 约束接收数据。

输出说明：见上文“输入输出”章节；输出必须使用标准引用或标准运行摘要。

配置说明：见上文“配置项草案”；配置只描述节点自身能力，不绑定具体 UI 控件。

数据流转方式：通过标准输入引用读取 TableRef，通过标准输出引用交付新的 TableRef。节点不直接读写主程序内部 Store，也不让主程序理解节点业务规则。

是否支持取消：支持；短耗时场景可在批处理边界检查取消，大表处理时按行或分批检查。

是否支持进度上报：支持低频进度上报，至少记录当前阶段、已处理数量和结果数量。

节点反馈：通过 NodeRun、RuntimeEvent 和结果摘要反馈开始、运行中、完成、失败、取消；摘要包含输入数量、输出数量、跳过数量和错误数量。

节点心跳：短耗时数据处理不强制主动心跳；处理大表或分批执行时按批次刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：中。

主要性能消耗点：按行或按列扫描、复制或生成 TableRef。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
