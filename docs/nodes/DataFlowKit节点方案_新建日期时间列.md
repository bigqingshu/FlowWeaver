# 节点方案：AddCurrentDateTimeColumnNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.current_datetime_column` / 新建日期时间列

FlowWeaver 暂定类型名：`AddCurrentDateTimeColumnNode`

节点方向：数据处理 / 运行时间标记

优先级：P0

当前状态：规划中，代码未实现

## 要解决的问题

为输入表新增一个日期时间字段，用于记录运行时间或逐行生成时间值。

用户看到的能力：配置输出字段和时间格式，运行后得到带时间列的新表。

不解决的内容：不读取外部时间服务，不修改系统时间，不写外部资源。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数不变，新增或覆盖一个日期时间字段。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `output_mode` | 生成新字段或覆盖已有字段 |
| `new_field` | 新字段名 |
| `target_field` | 覆盖目标字段 |
| `time_mode` | 整次运行固定同一时间、逐行实时获取 |
| `format_mode` | 占位符模板、strftime 模板 |
| `template` | 占位符模板 |
| `strftime_template` | Python strftime 模板 |

## 数据契约

节点应明确运行时间来源。

第一版建议默认“整次运行固定同一时间”，保证同一次 NodeRun 输出稳定可复核。

## 执行模式

支持普通运行和预览运行。

支持取消：低优先级。

支持进度：大表可按批次上报。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要。

## 主程序交互边界

节点通过运行上下文获取当前时间或由节点内部注入时钟，不访问主程序内部对象。

不要求 `WorkflowRunProcess` 为本节点提供专用时间逻辑。

## 运行记录

结果摘要建议包含输出字段、时间模式、格式模板、生成行数。

`NodeRun.current_stage` 可记录“生成时间值”“发布输出表”。

## 验收方式

固定时间模式下，同一次运行所有行时间值一致。

逐行模式下，时间格式正确且可取消。

格式模板非法时返回清晰错误。

## 实现前置依赖

需要通用表格节点执行入口。

需要可测试的时间来源注入。
