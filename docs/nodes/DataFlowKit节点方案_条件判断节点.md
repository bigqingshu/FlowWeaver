# 节点方案：ConditionFlagNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.condition_check` / 条件判断节点

FlowWeaver 暂定类型名：`ConditionFlagNode`

节点方向：流程控制 / 条件结果

优先级：P4

当前状态：规划中，代码未实现

## 要解决的问题

根据输入表状态或字段值生成一个条件结果，供后续条件跳转或条件边使用。

用户看到的能力：配置条件表达式，运行后得到 TRUE/FALSE 或自定义标志。

不解决的内容：第一版可只输出条件结果，不直接驱动调度跳转。

## 输入输出

输入：一个 `TableRef`。

输出：可透传输入 `TableRef`，并生成条件结果 DataRef 或运行上下文标志。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `flag_name` | 条件标志名 |
| `source_type` | 当前表等来源 |
| `condition_type` | 表行数、字段是否存在、字段值、空值数量、包含值数量 |
| `field` | 字段 |
| `op` | 操作符 |
| `value` | 比较值 |
| `value_source` | 比较值来源，固定值、字段值、指定单元格、表达式 |
| `value_field` | 字段值来源 |
| `case_sensitive` | 是否区分大小写 |
| `true_value` / `false_value` | 成立和不成立输出值 |

## 动态值来源补充

条件判断节点需要支持字段对字段判断，而不是只支持字段对固定值判断。

实际使用中可能需要：

```text
判断字段 A 是否等于字段 B。
判断字段 A 是否包含字段 B 当前行值。
判断字段 A 是否大于由字段 B 和字段 C 计算出的值。
```

建议第一版值来源至少支持：

```text
固定值：value 来自配置。
字段值：value 来自当前行指定字段。
指定单元格：value 来自固定行字段。
```

表达式值可后置，用于复合计算。

## 数据契约

条件结果应是标准 DataRef 或可记录的 NodeTaskResult 摘要，不建议只写进进程内 dict。

当比较值来自字段时，条件判断应明确聚合语义，例如任意行满足、全部行满足、首行判断或按行输出结果。

如果后续驱动调度，需要与条件边模型对齐。

## 执行模式

支持普通运行和预览运行。

支持取消：低优先级。

支持进度：按条件类型决定，扫描字段值时可上报。

## 副作用与确认

副作用与外部资源说明：无外部副作用；只生成条件结果。

是否需要用户确认：不需要。

## 主程序交互边界

节点不直接要求调度器跳转。

第一版建议作为数据/结果节点实现，后续再接入条件边。

## 运行记录

结果摘要建议包含条件类型、实际值、判断结果、输出标志。

RuntimeEvent 可记录条件判断结果。

## 验收方式

表行数、字段是否存在、字段值条件均可正确判断。

字段值条件支持固定值和字段值两类比较值来源。

字段不存在时按条件类型返回错误或 FALSE。

结果可被后续节点读取。

## 实现前置依赖

需要 DataRef 或条件结果引用模型。

如果驱动调度，需要条件边模型。

输出说明：见上文“输入输出”章节；输出必须使用标准引用或标准运行摘要。

配置说明：见上文“配置项草案”；配置只描述节点自身能力，不绑定具体 UI 控件。

数据流转方式：只产出标准控制状态或条件结果；真实跳转必须由通用调度能力解释，不让主程序为单个跳转节点写专用分支。

是否支持取消：支持；通常为短耗时节点，执行前后检查取消即可。

是否支持进度上报：支持基础阶段进度，通常只需要开始、判断完成和结果摘要。

节点反馈：反馈条件求值、命中分支、目标锚点、跳过原因、失败和取消。

节点心跳：短耗时控制节点不强制主动心跳；复杂条件或等待调度时可刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：后台极简模式保留开始、判断结果、目标锚点、失败和取消，不记录高频细节。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：低到中。

主要性能消耗点：条件求值、锚点解析和调度状态记录。

失败场景：条件配置不完整、目标锚点不存在、表达式非法、流程图形成不可执行路径。

失败提示：提示缺失的条件字段、目标锚点、非法表达式和当前可用锚点列表。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
