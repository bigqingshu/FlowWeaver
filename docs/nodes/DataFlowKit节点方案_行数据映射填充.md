# 节点方案：UnpivotRowsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.row_data_mapping` / 行数据映射填充

FlowWeaver 暂定类型名：`UnpivotRowsNode`

节点方向：数据处理 / 行展开

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

将一行中的多个值字段展开为多行，并保留指定字段作为上下文。

用户看到的能力：选择要展开的值字段和保留字段，运行后得到“长表”结构。

不解决的内容：不做复杂透视表聚合，不跨表映射，不写外部资源。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数可能增加，输出字段由保留字段、值字段、来源字段、原始行号和状态字段组成。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `mode` | 按行取值展开 |
| `start_row` / `end_mode` / `count` / `end_row` | 行范围 |
| `value_fields` | 要展开的值字段 |
| `keep_fields` | 保留字段 |
| `output_value_field` | 输出值字段名 |
| `output_source_field` / `source_field_name` | 是否输出来源字段名 |
| `output_original_row` / `original_row_field` | 是否输出原始行号 |
| `output_status` / `status_field` | 是否输出状态 |
| `empty_mode` / `empty_fixed` | 空值处理 |
| `trim_value` | 是否清理值空白 |

## 数据契约

节点对每个输入行遍历 `value_fields`，按空值策略生成 0 到多条输出行。

结果摘要必须记录输入行数、输出行数和跳过空值数量。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持，记录输入行处理进度和输出行数。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出；属于行数变化节点。

是否需要用户确认：不需要，建议预览。

## 主程序交互边界

节点只输出新的 `TableRef`。

主程序不理解行展开语义，也不为输出行数增加做专用处理。

## 运行记录

结果摘要建议包含展开字段数、输入行数、输出行数、跳过空值数量。

失败提示应指出缺失值字段、保留字段或输出字段冲突。

## 验收方式

多值字段可正确展开为多行。

保留字段在每条展开行中保持一致。

空值策略可跳过或写固定值。

## 实现前置依赖

需要通用表格节点执行入口。

需要输出行数上限策略。
