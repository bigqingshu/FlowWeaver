# 节点方案：WriteSelectedColumnsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.selected_columns_write` / 选定列写入指定表

FlowWeaver 暂定类型名：`WriteSelectedColumnsNode`

节点方向：输出 / 表写入

优先级：P2/P3

当前状态：规划中，代码未实现

## 要解决的问题

选择输入表中的部分字段，写入目标运行表、命名输出或外部数据库表。

用户看到的能力：选择字段和目标表，预览写入计划，确认后执行。

不解决的内容：第一版不做复杂数据库迁移，不默认允许后台无感写库。

## 输入输出

输入：一个 `TableRef`。

输出：一个状态 `TableRef` 或透传输入表。

写入目标可分为运行内目标和外部数据库目标。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `source_type` | 当前工作流表、SQLite 表、运行内命名表 |
| `selected_fields` | 选定字段 |
| `target_type` | 运行内目标、SQLite 表等 |
| `target_table` / `target_transit_table` | 目标名称 |
| `write_mode` | 新建字段、覆盖、追加等 |
| `field_name_mode` | 使用原字段名、添加前后缀、映射 |
| `field_mappings` | 字段映射 |
| `overwrite_rule` | 覆盖规则 |
| `enable_write` | 是否允许真实写入 |
| `backup_before_write` | 写入前备份 |

## 数据契约

预览运行只生成写入计划和状态表。

SQLite 等外部写入需要正式运行、用户确认和 `enable_write=true` 同时满足。

## 执行模式

支持预览运行。

正式写入后置实现。

支持取消：必须支持，尤其是批量写入。

支持进度：必须支持，记录写入行数和字段数。

## 副作用与确认

副作用与外部资源说明：可写运行数据或外部数据库。

是否需要用户确认：运行内写入视策略决定；SQLite 写入必须确认。

## 主程序交互边界

节点通过通用上下文请求写运行内目标或外部目标。

主程序不理解字段写入模式，不做节点专用分支。

不进入权限审计链路，改用副作用说明和确认机制。

## 运行记录

结果摘要建议包含目标类型、目标表、选定字段数、写入行数、跳过行数、是否真实写入。

失败提示应指出目标不可访问、字段缺失、备份失败或写入冲突。

## 验收方式

预览模式不会写 SQLite。

运行内目标写入可生成可查询的命名输出或状态表。

SQLite 写入在未确认或 `enable_write=false` 时被跳过并记录。

## 实现前置依赖

需要目标表抽象。

需要用户确认机制。

需要外部写入状态表约定。
