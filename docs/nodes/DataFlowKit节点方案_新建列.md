# 节点方案：AddColumnsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.new_columns` / 新建列

FlowWeaver 暂定类型名：`AddColumnsNode`

节点方向：数据处理 / 表结构变换

优先级：P0

当前状态：规划中，代码未实现

## 要解决的问题

为输入表新增一个或多个字段，并按配置写入默认值或空值。

用户看到的能力：选择一张输入表，填写新字段列表，运行后得到包含新增字段的新表。

不解决的内容：不原地修改输入 `TableRef`，不写外部文件或数据库，不处理跨表字段映射。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出表保留原字段和原行顺序，并追加或按后续扩展规则插入新字段。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `columns_text` | 新字段列表，每行一个字段 |
| `value_mode` | 统一默认值、按字段配置值、空值、同行字段值、表达式值 |
| `default_value` | 统一默认值 |
| `default_value_source` | 默认值来源，固定值、同行字段、指定单元格、表达式 |
| `source_field` | 同行字段值模式下的来源字段 |
| `conflict_mode` | 自动改名、跳过、覆盖、报错 |
| `strip_column_name` | 是否清理字段名前后空白 |
| `allow_empty_name` | 是否允许空字段名 |

## 动态值来源补充

新建列不应只支持固定默认值。

实际使用中可能需要：

```text
新建“处理后名称”字段，初始值来自当前行“原名称”字段。
新建“匹配值”字段，初始值来自当前行某个固定列。
新建“计算标记”字段，初始值由当前行字段计算得到。
```

建议第一版值来源至少支持：

```text
固定值：所有新增字段写入同一个默认值。
按字段固定值：每个新增字段在 columns_text 中配置自己的固定值。
同行字段：新增字段初始值来自当前行指定字段。
指定单元格：新增字段初始值来自固定行字段。
```

表达式值可后置，用于按当前行计算新字段初始值。

## 数据契约

节点只读取输入 `TableRef` 的 schema 和 rows，输出新的 schema 和 rows。

当默认值来源是同行字段或指定单元格时，节点必须逐行解析值来源，而不是只在配置阶段解析一次。

字段冲突策略必须在节点内部完成，并在结果摘要里记录新增、跳过或改名字段。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持，按行处理大表时检查取消。

支持进度：建议支持，至少按行批次上报。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要。

## 主程序交互边界

节点只读取 `NodeTask.config` 和 `input_refs`，通过通用运行上下文读取输入表并发布输出表。

不直接调用 `EngineHost`、`WorkflowRunProcess`、`MainWindowViewModel` 或 `RuntimeStore` 内部结构。

不要求主程序为 `AddColumnsNode` 增加专用分支。

## 运行记录

`NodeRun.current_stage` 可记录“读取输入表”“新增字段”“发布输出表”。

`NodeTaskResult.output_refs` 返回新表引用。

结果摘要建议包含新增字段数、跳过字段数、改名字段数、输出行数。

## 验收方式

给定两列两行输入表，新增两个字段后输出 schema 和行数正确。

字段重名时按 `conflict_mode` 得到预期行为。

默认值来自同行字段时，新字段每行值与来源字段对应行一致。

输入字段为空、配置字段为空、字段名非法时返回可理解错误。

## 实现前置依赖

需要通用表格节点执行入口和 `TableRef` 读写 helper。

需要节点 config schema 支持多行文本、枚举和布尔字段。

性能影响等级：中。

主要性能消耗点：按行或按列扫描、复制或生成 TableRef。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
