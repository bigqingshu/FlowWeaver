# 节点方案：NumericColumnOperationNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.numeric_column` / 列数字运算

FlowWeaver 暂定类型名：`NumericColumnOperationNode`

节点方向：数据处理 / 数值计算

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

对指定字段执行加、减、乘、除、序号、取整、四舍五入等数值运算。

用户看到的能力：选择目标字段和运算方式，配置操作数来源，运行后得到计算结果字段。

不解决的内容：不做任意代码表达式，不跨表聚合，不提供完整公式引擎。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数不变，可新增结果字段或覆盖目标字段。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `target_field` | 目标字段 |
| `operation` | 加、减、乘、除、序号、取整、四舍五入 |
| `operand_source` | 固定值、字段值、行号、序列 |
| `operand_value` | 固定操作数 |
| `operand_field` | 操作数字段 |
| `output_mode` / `output_field` | 输出方式和输出字段 |
| `non_number_policy` | 非数字处理 |
| `divide_zero_policy` | 除零处理 |
| `decimal_places` | 小数位数 |
| `range_mode` | 全部行、指定范围、参考字段非空行 |

## 数据契约

节点逐行读取目标值和操作数，按策略输出数字或错误替代值。

非数字和除零必须按配置处理，不应导致整张表不可理解地失败。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持，记录处理行数和异常值数量。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要。

## 主程序交互边界

节点只依赖通用表格读写能力。

不把数值运算符写进主程序调度层。

## 运行记录

结果摘要建议包含运算方式、目标字段、输出字段、非数字数量、除零数量。

失败提示应明确字段不存在、操作数非法或小数位配置错误。

## 验收方式

固定值和字段值操作数均可计算。

除零和非数字按配置处理。

指定范围只影响范围内行。

## 实现前置依赖

需要数字解析和格式化 helper。

需要通用表格节点执行入口。
