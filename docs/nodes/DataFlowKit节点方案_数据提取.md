# 节点方案：ExtractTextNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.extract` / 数据提取

FlowWeaver 暂定类型名：`ExtractTextNode`

节点方向：数据处理 / 文本提取

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

从指定字段中按规则提取文本，并写入新字段或覆盖字段。

用户看到的能力：选择源字段，选择正则、固定位置、分隔符、关键字之间等提取方式，运行后得到提取结果。

不解决的内容：不做 OCR，不读取外部文件，不执行脚本表达式。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数不变，可新增结果字段或覆盖源字段。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `source_field` | 源字段 |
| `method` | 正则、固定位置、左/右 N 位、分隔符、关键字之间、前后缀删除 |
| `output_mode` | 生成新字段、覆盖源字段、覆盖已有字段 |
| `new_field` | 新字段名 |
| `unmatched_mode` | 留空、保留原值、固定值、跳过该行 |
| `regex_pattern` / `regex_group` | 正则表达式和分组 |
| `start_pos` / `extract_len` / `position_base` | 固定位置参数 |
| `delimiter` / `part_index` | 分隔符提取参数 |
| `before_key` / `after_key` | 关键字之间提取参数 |
| `strip_result` | 是否清理结果空白 |
| `rule_value_source` | 提取规则来源，固定值、同行字段、指定单元格、表达式 |
| `unmatched_value_source` | 未匹配填充值来源，固定值、同行字段、表达式 |

## 动态值来源补充

本节点不应只假设正则、分隔符、前后关键字都来自固定配置。

实际使用中可能需要：

```text
每一行用本行的“前关键字”字段和“后关键字”字段提取内容。
每一行用本行的“正则模板”字段提取源字段。
未匹配时用本行的“兜底值”字段填充结果。
```

建议第一版值来源至少支持：

```text
固定值：正则、分隔符、关键字、未匹配固定值来自配置。
同行字段：规则值或未匹配兜底值来自当前行字段。
指定单元格：规则值来自某个固定行字段。
```

表达式值来源可后置，用于拼接动态正则或动态关键字。

## 数据契约

节点逐行读取源字段并计算提取结果。

如果提取规则来自字段，应在每一行执行前解析该行规则，并把规则为空、规则非法、字段缺失等情况写入结果摘要或状态字段。

未匹配策略必须明确记录，避免用户误以为所有行都成功提取。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持，记录已处理行数、匹配行数、未匹配行数。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要，正则提取建议预览。

## 主程序交互边界

节点不调用 UI 提取窗口。

提取方法和参数全部由 config 表达，主程序不理解具体提取算法。

## 运行记录

结果摘要建议包含提取方法、输出字段、匹配行数、未匹配行数、错误摘要。

失败提示应明确源字段、方法和非法参数。

## 验收方式

正则分组、固定位置和分隔符提取均可覆盖。

正则、分隔符或关键字来自同行字段时，每行可使用不同提取规则。

未匹配时按 `unmatched_mode` 输出。

源字段不存在或参数非法时返回清晰错误。

## 实现前置依赖

需要通用表格节点执行入口。

需要配置 schema 支持动态字段显示，或第一版先使用展开式通用表单。

节点心跳：短耗时数据处理不强制主动心跳；处理大表或分批执行时按批次刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：中。

主要性能消耗点：按行或按列扫描、复制或生成 TableRef。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
