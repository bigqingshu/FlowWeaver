# 节点方案：DeleteRowsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.delete_rows` / 删除行

FlowWeaver 暂定类型名：`DeleteRowsNode`

节点方向：数据处理 / 行过滤

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

按行号、行号范围、条件或空行规则从输入表中过滤掉指定行。

用户看到的能力：配置删除条件，运行后得到行数减少的新表。

不解决的内容：不删除外部数据库记录，不原地修改输入表。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出字段不变，行数可能减少。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `delete_mode` | 按行号列表、行号范围、条件、空行 |
| `row_spec` | 行号列表 |
| `start_row` / `end_row` | 行号范围 |
| `condition_field` | 条件字段 |
| `condition_op` | 条件操作 |
| `condition_value` | 条件值 |
| `condition_value_source` | 条件值来源，固定值、同行字段、指定单元格、表达式 |
| `condition_value_field` | 条件值来自字段时的来源字段 |
| `case_sensitive` | 是否区分大小写 |
| `empty_mode` / `empty_field` | 空行判断规则 |

## 动态值来源补充

按条件删除不能只支持固定条件值。

实际使用中可能需要：

```text
删除“字段 A 包含字段 B 当前行值”的行。
删除“字段 A 等于字段 C 当前行值”的行。
删除“字段 A 小于某个由当前行字段计算出来的值”的行。
```

建议第一版值来源至少支持：

```text
固定值：condition_value 来自手动输入。
同行字段：condition_value 来自当前行另一个字段。
指定单元格：condition_value 来自固定行字段。
```

表达式值可后置，用于按当前行计算比较值。

## 数据契约

节点根据配置计算要删除的行集合，并输出保留行。

如果条件值来自字段，节点必须对每一行分别解析比较值。

结果摘要必须记录删除行数和保留行数。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持，记录扫描行数和命中删除行数。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出；属于行数减少节点。

是否需要用户确认：不需要，建议预览。

## 主程序交互边界

节点不让主程序理解删除条件。

不要求 WorkflowRunProcess 对行删除做专用处理。

## 运行记录

结果摘要建议包含删除模式、删除行数、保留行数、条件摘要。

失败提示应指出字段不存在、行号非法或条件操作不支持。

## 验收方式

行号列表和范围删除正确。

条件删除可处理包含、等于、大小比较和空值。

条件值来自同行字段时，可以按行执行字段对字段比较。

空行删除符合配置。

## 实现前置依赖

可复用或扩展现有 `FilterRowsNode` 条件逻辑。

需要通用表格节点执行入口。
