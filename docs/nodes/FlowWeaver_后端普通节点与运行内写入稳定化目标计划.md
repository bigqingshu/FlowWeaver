# FlowWeaver 后端普通节点与运行内写入稳定化目标计划

更新时间：2026-07-07

## 目标定位

本目标用于在进入条件跳转、循环、节点组和子工作流之前，先把当前后端普通节点和运行内写入节点稳定下来。

当前阶段不进入复杂调度语义，不实现外部数据库真实写入，不打开插件真实执行。

## 执行范围

| 范围 | 当前策略 |
|---|---|
| 普通表处理节点 | 保持现有 handler、schema、批量读写和测试覆盖，优先补边界测试 |
| 运行内写入节点 | 继续限制在 `run_table` / `memory_table`，通过显式 `enable_write=true` 打开 |
| 外部写入目标 | SQLite / 外部数据库继续输出状态表并跳过真实写入 |
| 外部资源节点 | 保留显式动作开关和状态表，不扩大副作用范围 |
| 调度类节点 | 暂不实现，等稳定基线完成后单独进入 |

## 稳定化检查项

| 检查项 | 要求 |
|---|---|
| 默认安全 | 默认不真实写外部资源，不后台无感写入 |
| 显式开关 | 运行内写入必须由 `enable_write=true` 或节点自己的显式动作开关触发 |
| 目标类型 | 真实写入只允许 `run_table` / `memory_table` |
| 写入模式 | `create` 已有目标时失败，`append` 结构不一致时失败 |
| 状态表 | 写入节点必须输出 `actual_write`、`affected_rows`、`skipped_rows`、`warnings`、`target_table_ref_id` |
| 文档状态 | 节点实现状态文档必须和代码能力一致 |
| 测试基线 | 后端节点执行测试和节点定义 API 测试必须通过 |

## 当前已补强内容

| 节点 | 补强内容 |
|---|---|
| `WriteSelectedColumnsNode` | 增加 `create` 目标已存在失败测试、`append` 结构不一致失败测试 |
| `WriteBackTableNode` | 增加运行内目标空值跳过统计测试、`create` 目标已存在失败测试、`append` 结构不一致失败测试 |

## 下一步顺序

1. 运行普通节点与写入节点相关定向测试。
2. 运行后端节点和 API schema 集成测试。
3. 如无回归，将稳定化测试与文档单独提交。
4. 稳定基线通过后，再进入控制流节点的最小语义分析。

## 暂不进入范围

- 条件跳转、无条件跳转、跳转锚点。
- 循环执行起点和循环判断回跳。
- 节点组与子工作流。
- 外部数据库真实写入。
- 插件真实发现器和执行 runner。
