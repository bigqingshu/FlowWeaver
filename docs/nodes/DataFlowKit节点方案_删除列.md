# 节点方案：DropColumnsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.delete_columns` / 删除列

FlowWeaver 暂定类型名：`DropColumnsNode`

节点方向：数据处理 / 表结构变换

优先级：P0

当前状态：规划中，代码未实现

## 要解决的问题

从输入表中移除指定字段，输出一张不包含这些字段的新表。

用户看到的能力：选择要删除的字段，运行后得到字段更少的新表。

不解决的内容：不删除外部数据库字段，不原地修改输入表，不处理字段级权限审计。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出表保留未删除字段的顺序和所有行。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `fields` | 要删除的字段列表 |
| `missing_policy` | 字段不存在时跳过、警告或报错 |

## 数据契约

节点读取输入表 schema，根据字段名过滤 schema 和 row 值。

至少保留 0 个字段是否允许，需要在节点方案实现前确定；第一版建议允许输出空字段表，但在结果摘要中提示。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：低优先级，大表可按批次上报。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要，建议 UI 预览字段变化。

## 主程序交互边界

节点只通过 `NodeTask.config`、`input_refs` 和通用表格读写能力工作。

不要求 `WorkflowRunProcess` 识别本节点类型。

不依赖 Avalonia 专用编辑器，字段选择器只是 UI 增强。

## 运行记录

`NodeTaskResult.output_refs` 返回新表引用。

结果摘要建议包含删除字段名、保留字段数、输出行数、缺失字段警告。

## 验收方式

删除存在字段后，输出 schema 不包含该字段。

删除多个字段后，未删除字段顺序保持不变。

字段不存在时按 `missing_policy` 返回预期结果。

## 实现前置依赖

需要通用表格节点执行入口。

需要 config schema 支持字段多选。

是否支持进度上报：支持低频进度上报，至少记录当前阶段、已处理数量和结果数量。

节点反馈：通过 NodeRun、RuntimeEvent 和结果摘要反馈开始、运行中、完成、失败、取消；摘要包含输入数量、输出数量、跳过数量和错误数量。

节点心跳：通常不强制主动心跳，开始和结束状态即可；大表结构复制时可按批次刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：低。

主要性能消耗点：主要是字段结构调整和表头元数据处理，数据复制量随表宽变化。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
