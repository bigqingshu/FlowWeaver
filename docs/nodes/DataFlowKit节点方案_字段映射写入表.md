# 节点方案：WriteBackTableNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.writeback` / 字段映射写入表

FlowWeaver 暂定类型名：`WriteBackTableNode`

节点方向：输出 / 外部数据库写回

优先级：P3

当前状态：规划中，代码未实现

## 要解决的问题

按匹配规则和字段映射，将当前表数据写回目标表。

用户看到的能力：配置匹配键、字段映射和覆盖策略，先预览写回动作，确认后执行数据库更新。

不解决的内容：不默认后台无感写库，不提供跨数据库事务，不做完整数据同步平台。

## 输入输出

输入：一个源 `TableRef`，以及目标表引用或数据库目标配置。

输出：一个写回状态 `TableRef`，可包含匹配结果、动作类型、错误信息。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `writeback_direction` | 写回方向 |
| `target_table` | 目标表 |
| `source_table` | 源表 |
| `use_match_rules` / `match_rules` | 是否使用匹配规则和规则列表 |
| `field_mappings` | 源字段到目标字段映射 |
| `overwrite_policy` | 覆盖策略 |
| `source_empty_policy` | 源空值处理 |
| `no_match_policy` / `multi_match_policy` | 无匹配、多匹配处理 |
| `duplicate_target_policy` | 目标重复处理 |
| `enable_write` | 是否允许真实写入 |
| `backup_before_write` | 写前备份 |
| `output_preview_table` | 是否输出预览表 |

## 数据契约

预览运行必须只生成动作计划，不修改目标表。

正式写入必须同时满足用户确认、正式运行和 `enable_write=true`。

## 执行模式

支持预览运行：必须支持。

正式写回：后置实现。

支持取消：必须支持。

支持进度：必须支持，记录匹配和写入进度。

## 副作用与确认

副作用与外部资源说明：写数据库或外部表，属于高风险外部副作用。

是否需要用户确认：必须确认。

## 主程序交互边界

节点负责生成写回计划和执行写回动作。

主程序不理解匹配规则和字段映射。

数据库写入通过通用外部资源/目标表能力承接，不通过权限审计模块。

## 运行记录

结果摘要建议包含目标表、匹配规则数、计划动作数、成功数、失败数、跳过数、是否备份。

失败提示应指出目标表不可访问、匹配冲突、字段缺失、事务失败或用户未确认。

## 验收方式

预览模式只输出动作计划。

未确认或 `enable_write=false` 时不会写库。

测试数据库写回成功时，状态表和目标表变化一致。

## 实现前置依赖

需要用户确认机制。

需要目标数据库抽象和事务边界。

需要备份策略和状态表协议。
