# 节点方案：FillCellsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.fill_value` / 填充值

FlowWeaver 暂定类型名：`FillCellsNode`

节点方向：数据处理 / 单元格填充

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

按方向、范围和覆盖规则为目标字段填充值。

用户看到的能力：选择字段、起始行、方向、来源值和结束条件，运行后得到填充后的新表。

不解决的内容：不修改输入表，不跨文件填充，不写外部资源。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出字段和行数通常不变，只改变目标范围内的单元格。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `target_field` | 目标字段 |
| `start_row` | 起始行 |
| `direction` | 向下、向上、向右、向左 |
| `value_source` | 手动输入值或字段/区域来源 |
| `manual_value` | 手动填充值 |
| `source_field` / `source_row` | 来源字段和来源行 |
| `start_row_mode` / `end_mode` | 起止范围策略 |
| `count` / `end_row` / `reference_field` | 结束条件 |
| `overwrite_rule` | 覆盖所有或只填空单元格 |

## 数据契约

节点先解析填充目标范围，再按覆盖规则写入输出表。

填充范围过大时应有上限，避免一次配置影响过多单元格。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持，记录目标单元格数和实际写入数。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要，范围较大时建议确认或预览。

## 主程序交互边界

节点内部解析范围和方向。

主程序只提供输入表和接收输出表，不理解填充业务。

## 运行记录

结果摘要建议包含目标字段、范围、方向、目标单元格数、实际写入数。

失败提示应指出字段不存在、行号非法或范围超过上限。

## 验收方式

向下填充和向上填充正确。

只填空单元格不会覆盖已有值。

范围越界时返回清晰错误。

## 实现前置依赖

需要通用表格节点执行入口。

需要单元格上限策略。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
