# 节点方案：AdvancedFilterRowsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.filter` / 高级筛选

FlowWeaver 暂定类型名：`AdvancedFilterRowsNode`

节点方向：数据处理 / 多条件筛选

优先级：P2

当前状态：规划中，代码未实现

## 要解决的问题

支持多条件、多表关联和输出字段控制的高级行筛选。

用户看到的能力：配置条件组、关联表、输出字段和结果上限，运行后得到筛选结果表。

不解决的内容：第一版不做完整 SQL 编辑器，不直接写数据库，不绑定 DataFlowKit 旧高级筛选窗口。

## 输入输出

输入：一个主 `TableRef`，可选一个或多个 lookup `TableRef`。

输出：一个新的 `TableRef`。

输出行数可能减少，字段可按 `output_fields` 控制。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `logic` | 主条件 AND / OR |
| `conditions` | 条件列表，需包含字段、操作符、值、值来源 |
| `extra_tables` | 额外表 |
| `join_rules` | 关联规则 |
| `join_logic` | 关联条件逻辑 |
| `output_fields` | 输出字段 |
| `result_limit` | 结果上限 |
| `max_intermediate` | 中间结果上限 |
| `remove_duplicates` | 是否去重 |

## 动态值来源补充

高级筛选必须把“条件值来源”作为一等配置，否则会退化成只能字段对固定值筛选。

实际使用中可能需要：

```text
当前表字段 A 等于当前表字段 B。
当前表字段 A 包含副表字段 C。
当前表字段 A 大于由当前行字段计算出的值。
```

建议每条 `conditions[]` 至少支持：

```text
value_source = 固定值：value 是手动输入。
value_source = 字段值：value 指向当前表或副表字段。
value_source = 指定单元格：value 指向某个固定行字段。
value_source = 表达式：value 是受限表达式，后置。
```

`join_rules[]` 本质上也应按字段对字段比较设计，不应只支持字段对固定值。

## 数据契约

节点根据条件 DSL 过滤主表，并可读取额外 `TableRef` 做关联判断。

条件 DSL 需要在每行或每个关联候选上解析值来源，确保字段对字段、当前行动态值和固定值都能表达。

第一版可先从单表高级条件开始，后续再扩展多表关联。

## 执行模式

支持普通运行和预览运行。

支持取消：必须支持，筛选和关联可能耗时。

支持进度：建议支持，记录扫描行数和输出行数。

## 副作用与确认

副作用与外部资源说明：无外部副作用；多表模式读取额外输入引用。

是否需要用户确认：不需要，建议预览。

## 主程序交互边界

节点不依赖旧 UI 筛选窗口。

条件 DSL、关联规则和输出字段都通过 config 表达。

主程序不理解筛选条件，只负责传递输入引用。

## 运行记录

结果摘要建议包含输入行数、输出行数、条件数量、关联表数量、是否截断。

失败提示应指出非法条件、缺失字段或超过中间结果上限。

## 验收方式

单表 AND/OR 条件可正确筛选。

单表字段对字段条件可正确筛选。

输出字段控制生效。

结果上限和中间结果上限生效。

## 实现前置依赖

需要条件 DSL 方案。

需要多输入 `TableRef` 或额外表引用机制。

可先复用现有 `FilterRowsNode` 作为第一阶段入口。

是否支持取消：必须支持；大表筛选、关联和结果构建阶段都要检查取消。

是否支持进度上报：必须支持低频进度，记录扫描行数、命中行数、关联候选数和截断状态。

节点反馈：通过 NodeRun、RuntimeEvent 和结果摘要反馈开始、运行中、完成、失败、取消；摘要包含输入数量、输出数量、跳过数量和错误数量。

节点心跳：大表或多表关联时必须保留低频心跳，通过 NodeRun.last_heartbeat 判断是否仍在运行。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：中到高。

主要性能消耗点：多条件逐行扫描、字段对字段比较、可选多表关联和中间结果控制。

失败场景：输入表不可读、字段不存在、条件 DSL 非法、关联规则冲突或中间结果超过上限。

失败提示：提示非法条件、缺失字段、超限配置和具体失败阶段。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
