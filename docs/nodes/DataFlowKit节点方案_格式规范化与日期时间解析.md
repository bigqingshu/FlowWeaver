# 节点方案：ParseDateTimeNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.datetime_format` / 格式规范化 / 日期时间解析

FlowWeaver 暂定类型名：`ParseDateTimeNode`

节点方向：数据处理 / 日期时间解析

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

把源字段中的日期、时间或日期时间文本解析成标准格式。

用户看到的能力：配置输入结构、日期顺序、年份规则和输出模板，运行后得到标准日期时间字段和可选状态字段。

不解决的内容：不调用外部时间服务，不修改时区数据库，不自动推断所有地区格式。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数不变，可新增标准化字段、状态字段或覆盖已有字段。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `source_field` | 日期源字段 |
| `time_source_field` | 独立时间字段 |
| `use_separate_time_field` | 是否使用独立时间字段 |
| `parse_type` | 日期、时间、日期时间 |
| `input_structure` | 固定位置、分隔符、自动识别常见格式 |
| `date_order` | 年-月-日、月-日-年、日-月-年 |
| `ambiguous_date_policy` | 歧义日期处理 |
| `year_rule` | 两位年份补全规则 |
| `output_template` / `time_output_template` / `datetime_output_template` | 输出模板 |
| `output_status` / `status_field` | 是否输出解析状态 |

## 数据契约

节点逐行解析源值，输出标准化文本或空值/固定值。

日期歧义和解析失败不应静默吞掉，至少应通过状态字段或结果摘要体现。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议支持，记录解析成功和失败数量。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要，歧义日期建议预览。

## 主程序交互边界

节点内部处理日期解析策略。

主程序只负责传入 config 和输入 `TableRef`，不为日期格式增加全局规则。

## 运行记录

结果摘要建议包含解析类型、输出字段、成功行数、失败行数、歧义行数。

失败提示应指出日期顺序、字段名和示例值。

## 验收方式

固定位置和分隔符格式可正确解析。

两位年份规则可控。

歧义日期按 `ambiguous_date_policy` 处理。

## 实现前置依赖

需要可测试的日期解析 helper。

需要通用表格节点执行入口。

性能影响等级：中。

主要性能消耗点：按行或按列扫描、复制或生成 TableRef。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
