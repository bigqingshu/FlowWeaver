# 节点方案：ReplaceTextNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.replace` / 批量替换

FlowWeaver 暂定类型名：`ReplaceTextNode`

节点方向：数据处理 / 文本替换

优先级：P1

当前状态：规划中，代码未实现

## 要解决的问题

按匹配规则批量替换指定字段中的文本内容。

用户看到的能力：选择目标字段，配置匹配值、替换值和匹配模式，运行后得到替换后的新表。

不解决的内容：不跨表替换，不写外部资源，不做全文索引级搜索替换。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数和字段数默认不变，只改变目标字段或指定输出字段的值。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `target_field` | 目标字段 |
| `match_mode` | 包含、完全相等、开头是、结尾是、正则匹配、为空、不为空 |
| `match_value` | 手动匹配值 |
| `replace_value` | 手动替换值 |
| `replace_mode` | 局部替换匹配字符串、整格替换 |
| `case_sensitive` | 是否区分大小写 |
| `replace_count` | 每格替换次数，0 表示不限制 |
| `skip_empty_match_value` | 是否跳过空匹配值 |
| `match_value_source` / `replace_value_source` | 手动输入或字段值 |
| `match_value_field` / `replace_value_field` | 字段来源 |
| `match_value_expression` / `replace_value_expression` | 后置扩展，按当前行计算匹配值或替换值 |

## 动态值来源补充

本节点需要明确支持“用当前数据替换当前数据”的场景。

典型例子：

```text
目标字段内容为 123。
匹配值来自当前行的字段 A，值为 3。
替换值来自当前行的字段 B，值为 4。
运行后目标字段变为 124。
```

建议第一版值来源至少支持：

```text
固定值：手动输入 match_value / replace_value。
同行字段：从当前处理行的指定字段读取匹配值或替换值。
指定行字段：从固定行号的指定字段读取值。
```

后续再扩展：

```text
表达式值：基于当前行字段计算结果。
上游节点结果：读取上游 DataRef 中的单值结果。
```

## 数据契约

节点读取目标字段，对每行单元格按规则计算输出值。

当匹配值或替换值来自字段时，必须按行解析值来源，不能在节点开始时只解析一次固定值。

正则模式下应捕获正则编译错误，并给出字段名和表达式摘要。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持，按行批次检查。

支持进度：建议支持，记录已处理行数和命中数量。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要，但正则或空匹配建议预览。

## 主程序交互边界

节点只依赖 `NodeTask.config`、`input_refs` 和通用表格读写能力。

不让主程序理解匹配模式，也不要求 UI 专用窗口作为执行前提。

## 运行记录

结果摘要建议包含目标字段、匹配模式、命中行数、替换单元格数、正则错误。

`NodeRun.current_stage` 可记录“匹配替换”“发布输出表”。

## 验收方式

包含、完全相等、正则模式均能正确替换。

匹配值和替换值都来自同行字段时，可以完成逐行不同规则的替换。

空匹配值在 `skip_empty_match_value=true` 时不执行危险替换。

字段不存在、正则非法时返回可理解错误。

## 实现前置依赖

需要通用表格节点执行入口。

需要配置 schema 支持枚举、字段选择和文本输入。

配置说明：见上文“配置项草案”；配置只描述节点自身能力，不绑定具体 UI 控件。

数据流转方式：通过标准输入引用读取 TableRef，通过标准输出引用交付新的 TableRef。节点不直接读写主程序内部 Store，也不让主程序理解节点业务规则。

是否支持取消：支持；短耗时场景可在批处理边界检查取消，大表处理时按行或分批检查。

是否支持进度上报：支持低频进度上报，至少记录当前阶段、已处理数量和结果数量。

节点反馈：通过 NodeRun、RuntimeEvent 和结果摘要反馈开始、运行中、完成、失败、取消；摘要包含输入数量、输出数量、跳过数量和错误数量。

节点心跳：短耗时数据处理不强制主动心跳；处理大表或分批执行时按批次刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：中。

主要性能消耗点：按行或按列扫描、复制或生成 TableRef。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
