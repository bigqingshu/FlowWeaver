# 节点方案：ListFilesNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.file_list` / 获取文件列表

FlowWeaver 暂定类型名：`ListFilesNode`

节点方向：文件处理 / 外部资源只读输入

优先级：P2

当前状态：规划中，代码未实现

## 要解决的问题

读取本地目录中的文件或文件夹清单，并输出为表格数据。

用户看到的能力：选择目录、递归选项、扩展名和数量上限，运行后得到文件列表表。

不解决的内容：不读取文件内容，不修改文件，不监控目录变化。

## 输入输出

输入：目录路径和过滤配置；不要求输入 `TableRef`。

输出：一个 `TableRef`，字段可包含文件名、完整路径、扩展名、大小、修改时间、是否目录等。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `directory` | 目录路径 |
| `recursive` | 是否递归 |
| `include_files` | 是否包含文件 |
| `include_dirs` | 是否包含文件夹 |
| `include_hidden` | 是否包含隐藏项 |
| `extensions` | 扩展名过滤 |
| `name_contains` | 文件名包含文本 |
| `glob_pattern` | glob 匹配模式 |
| `max_files` | 最大返回数量 |

## 数据契约

节点扫描目录并生成表格输出。

输出字段第一版应稳定，避免后续节点引用不可靠。

## 执行模式

支持普通运行和预览运行。

支持取消：必须支持，递归目录可能耗时。

支持进度：建议支持，记录扫描目录数和命中数量。

## 副作用与确认

副作用与外部资源说明：只读外部文件系统。

是否需要用户确认：建议确认，尤其是后台运行或扫描大目录时。

## 主程序交互边界

节点自行访问文件系统，不要求主程序提供目录扫描专用服务。

主程序只记录运行状态和输出 `TableRef`。

不进入权限审计链路。

## 运行记录

结果摘要建议包含目录路径、递归选项、命中数量、截断数量、扫描耗时。

失败提示应指出目录不存在、不可访问或过滤配置非法。

## 验收方式

可列出指定目录文件。

递归、扩展名和隐藏文件选项生效。

超过 `max_files` 时输出被截断并记录提示。

## 实现前置依赖

需要支持无输入表的 Source 节点。

需要外部资源确认机制或运行来源策略。

节点名称：获取文件列表。

节点定位：文件处理 / 外部资源只读输入。

优先级：P2。

当前状态：规划中，代码未实现。

要解决的问题：见上文“要解决的问题”章节。

用户看到的能力：见上文“用户看到的能力”描述。

注册参数：
- node_type：ListFilesNode

- ui_visibility：visible
- enabled：规划期为 false；实现和验收后再按节点成熟度设为 true。

- output_ports：一个文件清单 TableRef 输出。
- implementation_ref：builtin.ListFilesNode（暂定内部执行入口，后续实现时绑定真实实现；不对普通 UI 暴露）。

输入说明：见上文“输入输出”章节；第一版按 input_ports 约束接收数据。

输出说明：见上文“输入输出”章节；输出必须使用标准引用或标准运行摘要。

配置说明：见上文“配置项草案”；配置只描述节点自身能力，不绑定具体 UI 控件。

数据流转方式：节点按配置只读访问外部文件系统，生成文件清单 TableRef 后通过标准输出交付；不直接写主程序 Store。

是否支持取消：必须支持；递归扫描目录时按目录或批次检查取消。

是否支持进度上报：必须支持低频进度，记录已扫描目录数、命中文件数和截断状态。

节点反馈：反馈扫描开始、当前目录阶段、命中数量、截断数量、完成、失败和取消。

节点心跳：递归或大目录扫描必须保留低频心跳，通过 NodeRun.last_heartbeat 判断是否仍在活动。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：后台运行只保留开始、结束、失败、取消、命中数量、截断数量和低频心跳，不记录每个文件的详细日志。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：中到高。

主要性能消耗点：外部文件系统递归扫描、路径过滤、文件属性读取和结果表构建。

失败场景：目录不存在、目录不可访问、过滤配置非法、扫描数量超过上限或外部文件系统响应慢。

失败提示：提示具体目录、过滤项、访问错误和是否因 max_files 被截断。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
