# 节点方案：CopyColumnNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.copy_column` / 复制列

FlowWeaver 暂定类型名：`CopyColumnNode`

节点方向：数据处理 / 表结构与内容变换

优先级：P0

当前状态：规划中，代码未实现

## 要解决的问题

将输入表的某个字段复制成新字段，或覆盖已有字段。

用户看到的能力：选择源字段和目标字段，运行后得到包含复制结果的新表。

不解决的内容：不跨表复制，不写外部表，不处理复杂表达式。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数不变，源字段值复制到目标字段。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `source_field` | 源字段 |
| `output_mode` | 生成新字段、覆盖已有字段 |
| `new_field` | 新字段名 |
| `target_field` | 覆盖模式下的目标字段 |
| `trim_value` | 是否清理值前后空白 |
| `empty_default` | 源值为空时的默认值 |

## 数据契约

节点读取源字段值，并在输出表中生成或覆盖目标字段。

第一版建议默认生成新字段，避免无意覆盖。

## 执行模式

支持普通运行和预览运行。

支持取消：建议支持。

支持进度：建议按行批次上报。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出。

是否需要用户确认：不需要。

## 主程序交互边界

节点只依赖 `NodeTask.config` 和输入 `TableRef`。

字段选择器由 UI 根据 schema 渲染，但不是节点执行前提。

不要求 `WorkflowRunProcess` 为本节点增加专用逻辑。

## 运行记录

结果摘要建议包含源字段、目标字段、输出行数、空值替换次数。

`NodeTaskResult.output_refs` 返回新表引用。

## 验收方式

源字段存在时，新字段值与源字段一致。

源字段不存在时返回清晰错误。

覆盖目标字段时只影响输出表，不影响输入表。

## 实现前置依赖

需要通用表格节点执行入口。

需要 config schema 支持字段单选和输出模式枚举。

是否支持取消：支持；短耗时场景可在批处理边界检查取消，大表处理时按行或分批检查。

是否支持进度上报：支持低频进度上报，至少记录当前阶段、已处理数量和结果数量。

节点心跳：短耗时数据处理不强制主动心跳；处理大表或分批执行时按批次刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：中。

主要性能消耗点：按行或按列扫描、复制或生成 TableRef。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
