# 节点方案：RenameColumnsNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.rename_columns` / 批量更改列名

FlowWeaver 暂定类型名：`RenameColumnsNode`

节点方向：数据处理 / 表结构变换

优先级：P0

当前状态：规划中，代码未实现

## 要解决的问题

按映射、前缀、后缀或查找替换规则批量调整字段名。

用户看到的能力：配置字段改名规则，运行后得到字段名已更新的新表。

不解决的内容：不修改外部数据库 schema，不处理字段级权限审计，不自动重写后续节点配置。

## 输入输出

输入：一个 `TableRef`。

输出：一个新的 `TableRef`。

输出行数和单元格值不变，只改变 schema 字段名。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `mode` | 手动映射、添加前缀、添加后缀、查找替换 |
| `mappings` | 原字段到新字段的映射 |
| `prefix` / `suffix` | 前缀或后缀 |
| `replace_match` / `replace_value` | 字段名查找替换 |
| `scope` | 全部字段或指定字段 |
| `scope_fields` | 指定字段列表 |
| `duplicate_policy` | 重名处理 |
| `missing_policy` | 缺失字段处理 |
| `trim_names` | 清理字段名前后空白 |

## 数据契约

节点根据输入 schema 生成新的字段名列表。

重名策略必须在节点内部确定，结果摘要记录自动改名、跳过或报错情况。

## 执行模式

支持普通运行和预览运行。

支持取消：低优先级。

支持进度：可不强制；大 schema 或大表可按批次上报。

## 副作用与确认

副作用与外部资源说明：无外部副作用，只写本次运行输出；属于 schema 变化节点。

是否需要用户确认：不需要，但建议 UI 展示改名前后对照。

## 主程序交互边界

节点不负责更新工作流下游节点配置。

主程序不因本节点识别字段改名语义，后续字段引用失败由对应节点自身校验。

不要求 `EngineHost` 或 `WorkflowRunProcess` 增加专用处理。

## 运行记录

结果摘要建议包含改名数量、跳过数量、冲突数量和字段映射摘要。

错误信息应明确指出冲突字段或缺失字段。

## 验收方式

手动映射模式可以正确改名。

前缀、后缀、查找替换模式只影响目标范围。

重名时按 `duplicate_policy` 返回预期结果。

## 实现前置依赖

需要结构化列表配置 schema。

需要通用表格节点执行入口。

配置说明：见上文“配置项草案”；配置只描述节点自身能力，不绑定具体 UI 控件。

数据流转方式：通过标准输入引用读取 TableRef，通过标准输出引用交付新的 TableRef。节点不直接读写主程序内部 Store，也不让主程序理解节点业务规则。

是否支持取消：支持；短耗时场景可在批处理边界检查取消，大表处理时按行或分批检查。

是否支持进度上报：支持低频进度上报，至少记录当前阶段、已处理数量和结果数量。

节点反馈：通过 NodeRun、RuntimeEvent 和结果摘要反馈开始、运行中、完成、失败、取消；摘要包含输入数量、输出数量、跳过数量和错误数量。

节点心跳：通常不强制主动心跳，开始和结束状态即可；大表结构复制时可按批次刷新 NodeRun.last_heartbeat。

心跳查询方式：通过 NodeRun.last_heartbeat、status、progress、current_stage、error 和 RuntimeEvent 查询节点活动状态。

后台极简模式：保留开始、结束、失败、取消和最终结果摘要；关闭逐行日志和高频 UI 刷新，大表场景保留低频进度和心跳。

外部资源与副作用：沿用本文“副作用与确认”章节；权限、审计、字段级追踪不写入默认节点方案。

性能影响等级：低。

主要性能消耗点：主要是字段结构调整和表头元数据处理，数据复制量随表宽变化。

失败场景：输入表不可读、字段不存在、配置不完整、数据类型不符合预期。

失败提示：提示缺失字段、非法配置项或失败阶段，并给出可修正的字段名、配置名和样例值。

验收方式：沿用本文“验收方式”章节，并补充校验运行记录、取消、进度、心跳和后台极简模式是否符合本节约束。
