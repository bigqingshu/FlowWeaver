# FlowWeaver 后端节点实施顺序

更新时间：2026-07-07

## 文档定位

本文件用于承接 DataFlowKit 节点进入 FlowWeaver 后端实现前的执行顺序规划。

当前 UI 侧正在进行大文件拆分，后端节点实现应优先保证主程序与节点能力之间的边界清晰，避免后续批量增加节点时把执行分派、表处理逻辑和节点业务逻辑混在一起。

## 总体目标

```text
先建立轻量节点执行框架。
再沉淀通用表格处理能力。
随后按纯表节点、输出节点、外部资源节点的顺序逐步实现。
配置字继续保持运行观测配置定位，与节点业务 config 分层处理。
```

## 阶段 1：后端节点 Handler 注册机制

### 目标

将当前内置表节点执行入口从集中 `if/elif` 分派，逐步调整为按 `node_type` 查找独立 handler。

### 修改方向

```text
增加节点 handler 协议。
增加内置节点 handler 注册表。
让 BuiltinTableNodeRunner 只负责查找和调用 handler。
每个 handler 只处理一个节点类型或一组高度相关的节点类型。
```

### 完成标志

```text
现有 GenerateTestTableNode、FilterRowsNode、AddColumnsNode、SaveMemoryTableNode 行为保持一致。
新增节点无需继续扩展 BuiltinTableNodeRunner 的业务分支。
单个节点的配置校验和执行逻辑能独立测试。
```

## 阶段 2：NodeRuntimeContext 节点运行上下文

### 目标

为节点提供统一运行上下文，让节点通过上下文访问输入、输出、取消、进度和运行期辅助能力。

### 修改方向

```text
在现有 NodeExecutionContext 基础上扩展或新增 NodeRuntimeContext。
上下文提供输入 TableRef 读取入口。
上下文提供输出 staging/publish 辅助入口。
上下文提供取消检查入口。
上下文预留进度上报入口。
上下文封装 RuntimeStore、RuntimeDataRegistry、TableProvider 等主程序对象。
```

### 完成标志

```text
节点 handler 不直接组合主程序内部对象。
节点 handler 只依赖 task、context 和自身配置模型。
表输出发布流程由上下文统一处理。
```

## 阶段 3：通用表格处理工具层

### 目标

在批量实现节点前，先沉淀表格节点共用能力，减少每个节点重复写 schema、行遍历、值转换和输出构造逻辑。

### 修改方向

```text
增加 schema 字段查找与校验工具。
增加列新增、删除、复制、移动、重命名的 schema 构造工具。
增加固定值、当前行字段值、选定列值的 ValueSource 解析工具。
增加文本匹配、局部替换、整值替换的通用函数。
增加数据类型转换与空值处理工具。
增加批量读写 chunk helper。
```

### 完成标志

```text
后续节点可以复用统一 ValueSource 表达。
批量替换、填充值、复制列、行数据映射等节点共用同一套取值逻辑。
大表处理默认走分批读取和分批写入。
```

## 阶段 4：配置 Schema 表达能力增强

### 目标

让后端节点定义能够表达 DataFlowKit 节点所需的复杂配置，同时继续服务 UI 通用 JSON/表单渲染。

### 修改方向

```text
为 config_schema 增加字段分组能力。
支持 array object 类型字段。
支持模式选择字段。
支持列选择器提示。
支持字段间条件显示提示。
支持中文显示文本与英文 key 分层。
支持 ValueSource 配置结构的统一描述。
```

### 完成标志

```text
节点定义能完整描述第一批纯表节点配置。
UI 通用编辑器可以读取 schema 并生成可用配置草稿。
后端可以基于 schema 或节点配置模型完成校验。
```

## 阶段 5：第一批纯表节点实现

### 目标

优先实现对主程序影响小、便于测试、复用表格工具层最多的节点。

### 建议批次

| 批次 | 节点 | 依赖能力 |
|---|---|---|
| 5.1 | 新建列、删除列、复制列、移动列 | schema 构造、chunk 读写 |
| 5.2 | 填充值、批量替换、区域填充 | ValueSource、匹配替换、类型转换 |
| 5.3 | 删除行、复制行、去重与重复数据处理 | 行谓词、行输出策略、统计摘要 |
| 5.4 | 高级筛选、数据提取、匹配值输出列名 | 条件表达式、列选择、输出 schema |
| 5.5 | 合并列、列数字运算、新建日期时间列、格式规范化与日期时间解析 | 多列取值、计算函数、日期解析 |

### 完成标志

```text
每个节点都有独立 handler。
每个节点都有后端定义和配置 schema。
每个节点至少覆盖成功路径、配置错误、空表、大表 chunk 路径。
```

## 阶段 6：输出与中转类节点实现

### 目标

在纯表节点稳定后，实现与运行期数据保存、表间写入相关的节点。

### 建议批次

| 批次 | 节点 | 依赖能力 |
|---|---|---|
| 6.1 | 保存中转数据 | 输出注册、辅助表角色、运行记录 |
| 6.2 | 选定列写入指定表 | 列选择、目标表 schema 校验、写入策略 |
| 6.3 | 字段映射写入表 | 字段映射、类型转换、写入结果摘要 |

### 完成标志

```text
输出节点的副作用边界由节点定义和结果摘要表达。
写入结果能返回 affected_rows、skipped_rows 或 warnings。
失败信息能落到统一节点错误结构。
```

## 阶段 7：结果与诊断信息收口

### 目标

统一节点执行结果中的影响行数、跳过行数、警告、指标和诊断信息，避免各节点自行定义不兼容 payload。

### 修改方向

```text
确认 NodeTaskResultModel 与 NodeResultModel 的职责边界。
定义节点 handler 返回的内部结果对象。
统一 affected_rows、skipped_rows、warnings、metrics 的来源。
将运行配置字对事件和诊断 payload 的过滤继续放在中央出口。
```

### 完成标志

```text
节点结果摘要字段稳定。
UI 和日志读取同一套节点结果语义。
运行配置字可以继续控制事件层面的输出粒度。
```

## 阶段 8：外部资源类节点

### 目标

在表格核心和输出节点稳定后，实现文件、目录、插件等需要外部资源边界的节点。

### 建议批次

| 批次 | 节点 | 依赖能力 |
|---|---|---|
| 8.1 | 获取文件列表 | 路径配置、文件元数据 schema、后台执行 |
| 8.2 | 批量重命名 | 文件操作计划、执行结果摘要、错误回滚策略 |
| 8.3 | 插件节点 | 插件发现、插件配置、执行隔离、结果转换 |

### 完成标志

```text
外部资源访问通过明确配置和结果摘要表达。
后台执行路径可复用。
插件节点不会影响普通内置节点执行链路。
```

## 推荐执行节奏

```text
第一轮：阶段 1 + 阶段 2，完成节点执行框架收口。
第二轮：阶段 3 + 阶段 4，完成表格工具和配置表达基础。
第三轮：阶段 5.1 + 5.2，验证纯表节点批量实现方式。
第四轮：阶段 5.3 到 5.5，补齐主要数据处理节点。
第五轮：阶段 6 + 阶段 7，完成输出和结果摘要收口。
第六轮：阶段 8，进入外部资源类节点。
```

## 第一轮最小目标

```text
新增节点 handler 协议。
新增内置节点 handler 注册。
迁移一个现有节点作为样例。
保留原有节点行为。
补充样例节点测试。
确认后续新节点可以按 handler 文件独立增加。
```
