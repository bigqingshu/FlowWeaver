# 节点方案：BatchRenameFilesNode

更新时间：2026-07-05

## 基本信息

DataFlowKit 来源：`core.batch_rename` / 批量重命名

FlowWeaver 暂定类型名：`BatchRenameFilesNode`

节点方向：文件处理 / 外部写入

优先级：P3

当前状态：规划中，代码未实现

## 要解决的问题

根据输入表中的路径字段和新名称字段批量重命名文件或文件夹，并输出执行状态表。

用户看到的能力：先预览重命名计划，确认后再真实执行。

不解决的内容：不做复杂文件同步，不跨机器操作，不提供回滚保证。

## 输入输出

输入：一个包含文件路径和新名称的 `TableRef`。

输出：一个 `TableRef` 状态表，包含原路径、新路径、状态、错误信息等。

## 配置项草案

| 字段 | 含义 |
|---|---|
| `path_field` | 原完整路径字段 |
| `new_name_field` | 新文件名字段 |
| `name_value_type` | 仅文件名或完整路径 |
| `new_path_field` | 输出新完整路径字段 |
| `status_field` | 状态字段 |
| `auto_append_ext` | 是否自动补扩展名 |
| `allow_dirs` | 是否允许重命名文件夹 |
| `create_target_dirs` | 是否创建目标目录 |
| `conflict_mode` | 目标已存在时处理 |
| `actual_rename` | 是否允许真实执行 |
| `write_log` / `log_path` | 是否写日志和日志路径 |

## 数据契约

预览运行只生成计划和状态，不修改文件。

正式运行也必须同时满足用户确认和 `actual_rename=true` 才允许真实重命名。

## 执行模式

支持预览运行：必须支持，且不得产生外部写入。

支持普通运行：后置实现。

支持取消：必须支持，取消后输出已处理状态摘要。

支持进度：必须支持，按文件数量上报。

## 副作用与确认

副作用与外部资源说明：写入外部文件系统，可能重命名文件、创建目录、写日志。

是否需要用户确认：必须确认。

## 主程序交互边界

节点负责文件操作生命周期和错误收集。

主程序不管理具体文件重命名事务，不为本节点增加专用分支。

高风险确认应通过通用确认机制表达，不通过权限审计模块。

## 运行记录

结果摘要建议包含计划数量、成功数量、跳过数量、失败数量、是否真实执行、日志路径。

`RuntimeEvent` 可记录开始、进度、完成、失败摘要。

## 验收方式

预览模式不会修改文件。

正式模式在确认且 `actual_rename=true` 时修改测试目录文件。

目标冲突、路径不存在、权限不足时输出状态表而不是中断全部。

## 实现前置依赖

需要用户确认机制。

需要外部文件写入节点的后台运行策略。

需要取消后状态输出约定。
